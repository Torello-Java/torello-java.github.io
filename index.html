JAVA-HTML .JAR — OVERVIEW



The JAVA-HTML library is Ralph Torello’s tooling for building, upgrading, and publishing high-quality static documentation sites. It powers a custom JavaDoc pipeline, rich HTML/JSON/CSS utilities, and a Chrome DevTools Protocol wrapper for automated browser work — all designed to run reliably in large builds and stay friendly to static hosting.

WHAT IT IS

  • A build-first approach to JavaDoc that inserts upgrades and external HTML without cluttering source files.
  • Fast, deterministic HTML/JSON/CSS utilities for parsing, transforming, and formatting documentation assets.
  • A lightweight CDP/WebSocket layer for scripted browser tasks (headless or headful).

KEY COMPONENTS

  • JDU — JAVADOC UPGRADER: Applies structured upgrades to plain Oracle JavaDoc output. Supports external HTML injection, link cleanup, layout tweaks, and consistent site-wide formatting while keeping sources clean.
  • TORELLO.HTML: An HTML parser and toolkit that produces editable node structures for precise transforms (lists, tables, anchors, attribution footers, line-length hygiene). Includes NodeSearch helpers to find and rewrite elements safely.
  • TORELLO.JSON: Lean JSON readers/writers (e.g., ReadJSON, boxed/primitive helpers) used across the toolchain for configs and content stitching.
  • CSS TOKENIZER / PARSERS: Tokenizer derived from CSS Syntax Level 3 pseudo-code; used to analyze and normalize styles in generated docs. (Higher-level elementization sits on top of the tokens.)
  • TORELLO.BROWSER (CDP WRAPPER): A Java API over Chrome DevTools Protocol via WebSockets. Core classes like BrowserConn, PageConn, and WebSocketSender enable scripted navigation, JS evaluation, event/error handling, and scraping pipelines.

WHY IT EXISTS

Produce static, cache-friendly, link-stable documentation you can publish anywhere (Cloudflare R2, GCS, GitHub Pages). The library favors clarity, repeatable builds, and zero surprise diffs over ad-hoc HTML hand edits.
