TORELLO.JAVA — JAVA-HTML .JAR LIBRARY


The JAVA-HTML library is a build-first toolchain for producing clean, static, cache-friendly documentation sites. It upgrades plain Oracle JavaDoc, provides tight HTML/JSON/CSS utilities, and ships a light Chrome DevTools Protocol wrapper for scripted browser work — all designed to stay deterministic across very large builds.

WHAT IT IS

  • JDU — JAVADOC UPGRADER — Structured post-processing for JavaDoc output: external HTML embedding, link corrections, layout polish, and consistent site-wide formatting without cluttering source files.

  • Torello.HTML — An HTML parser + toolkit that turns documents into editable node trees. Paired with Torello.HTML.NodeSearch for safe, targeted transforms (lists, tables, anchors, attribution, line-length hygiene).

  • Torello.JSON — Lean readers/writers (e.g., ReadJSON, boxed/primitive helpers) for configuration and content stitching across the toolchain.

  • CSS TOKENIZER / PARSERS — Tokenizer derived from CSS Syntax Level 3 pseudo-code, with higher-level elementization layered on top for analysis and normalization of generated styles.

  • Torello.Browser — CDP/WEBSOCKET WRAPPER — Java bindings over Chrome DevTools Protocol via WebSockets (e.g., BrowserConn, PageConn, WebSocketSender). Enables scripted navigation, JS evaluation, event/error handling, and scraping pipelines (headless or headful).

WHY IT EXISTS

Static hosting is reliable, cheap, and fast. This library favors clarity and repeatable builds over ad-hoc HTML edits, so you can publish to Cloudflare R2, GCS, or GitHub Pages with minimal moving parts — and get identical output every time.
